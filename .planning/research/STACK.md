# Project Research: Stack

**Project:** ThreeZinc Audio Studio
**Date:** 2026-04-27
**Source:** `AUDIO_APP_PRD.md`, `Audio.md`, `threezinc_design_system (1).md`, Fal/Gemini docs reviewed during PRD creation.

## Recommended Stack

### Application

- Next.js 14+ with App Router.
- TypeScript strict mode.
- React client components for the studio surface.
- Server route handlers for provider calls.

### Styling and UI

- Tailwind CSS.
- shadcn/ui for standard controls: Button, Slider, Select, Tabs, Tooltip, Dialog/Sheet, ScrollArea, Separator, Input, Textarea, Badge.
- lucide-react icons.
- Framer Motion for subtle UI entrance and bottom player transitions.
- ThreeZinc design tokens from `threezinc_design_system (1).md`.

### Audio

- Native `HTMLAudioElement` for preview and generated audio playback.
- One global audio manager to prevent simultaneous playback.
- Static preview WAV files in `public/previews/`.
- Generated audio requested as provider `mp3` by default.

### Provider

- `@fal-ai/client` from a server route only.
- Model: `fal-ai/gemini-3.1-flash-tts`.
- Environment variable: `FAL_KEY`.
- Request fields: `prompt`, `style_instructions`, `voice`, `language_code`, `speakers`, `output_format`.

### Persistence

- IndexedDB preferred for scripts, generation history metadata, and settings.
- localStorage acceptable for a minimal first pass or settings.
- No SQLite in V1.
- No local generated audio file writes in V1.
- Storage hidden behind `lib/client-store.ts`.

## Initial Dependency Set

Production:

- `@fal-ai/client`
- `framer-motion`
- `lucide-react`
- shadcn/ui dependencies as installed by the CLI
- optional tiny IndexedDB helper only if it reduces complexity; otherwise use native IndexedDB wrapper

Development:

- Next.js defaults
- TypeScript
- ESLint
- Tailwind CSS

## Environment Variables

```txt
FAL_KEY=your_fal_api_key_here
```

Rules:

- `.env.local` is never committed.
- `.env.example` is committed.
- Server route returns a setup error if `FAL_KEY` is missing.

## Open Technical Checks Before Coding

- Confirm current Next.js version scaffold generated by `create-next-app`.
- Confirm shadcn/ui install path and component aliases.
- Confirm Fal response shape in real local test.
- Confirm whether provider returns CORS-downloadable URLs or whether download needs a client fetch/blob fallback.

## Recommendation

Use a normal Next.js app scaffold in the workspace root. Keep app-specific implementation under `app/`, `components/tts/`, `config/`, `hooks/`, `lib/`, and `types/`. Do not build a monorepo or library package for V1.
