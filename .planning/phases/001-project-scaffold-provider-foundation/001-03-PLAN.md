<task type="auto">
  <name>Implement server-side Fal generation route</name>
  <objective>Create `POST /api/tts/generate` with validation, retry, structured errors, and manual backend validation instructions.</objective>
  <context>
    This is the only provider route in V1. It must be safe to call from the browser without exposing secrets.
  </context>
  <steps>
    <step id="1">Create `app/api/tts/generate/route.ts`.</step>
    <step id="2">Parse request JSON safely and return structured `BAD_JSON` or `INVALID_REQUEST` errors.</step>
    <step id="3">Use validation helpers from `lib/tts-validation.ts`.</step>
    <step id="4">If `provider` is `clone` or `openai`, return status 501 with structured `PROVIDER_NOT_IMPLEMENTED` error.</step>
    <step id="5">If `FAL_KEY` is missing, return structured setup error. Do not crash.</step>
    <step id="6">Configure `fal.config({ credentials: process.env.FAL_KEY })` server-side only.</step>
    <step id="7">For single mode, send `prompt`, optional `style_instructions`, `voice`, optional `language_code`, and `output_format: "mp3"`.</step>
    <step id="8">For multi mode, require exactly two speakers and send provider-compatible `speakers` config.</step>
    <step id="9">Retry once for transient provider failures. Keep logs server-side.</step>
    <step id="10">Validate returned audio shape before returning success.</step>
    <step id="11">Return success with `audio`, optional `requestId`, `characterCount`, and `estimatedCost`.</step>
    <step id="12">Create `docs/backend-validation.md` with sample PowerShell/curl commands for bad request, missing key, and real generation when key exists.</step>
  </steps>
  <threat_model>
    <asset>FAL_KEY</asset>
    <threat>Secret exposure in client bundle or committed files.</threat>
    <mitigation>Server route only, `.env.example` only, no client imports of provider SDK.</mitigation>
    <asset>Provider billing</asset>
    <threat>Malformed or repeated requests create unexpected costs.</threat>
    <mitigation>Validate prompt/limits, retry only once, structured errors.</mitigation>
  </threat_model>
  <must_haves>
    <truth>`FAL_KEY` never appears in client code.</truth>
    <truth>No provider SDK import in client components/hooks.</truth>
    <truth>Every error response has `error.code`, `error.message`, and `error.retryable`.</truth>
    <truth>Generated audio source of truth is provider URL plus metadata only.</truth>
  </must_haves>
  <verify>
    <command>npm run lint</command>
    <command>npm run typecheck</command>
    <command>npm run build</command>
    <check>Bad request returns structured JSON.</check>
    <check>Missing `FAL_KEY` returns setup error.</check>
    <check>If `FAL_KEY` exists, sample single-speaker request returns audio URL.</check>
    <check>`app/api/tts/preview/route.ts` still does not exist.</check>
  </verify>
  <done>Backend foundation is validated and Phase 2 can build UI against the generate route.</done>
</task>
